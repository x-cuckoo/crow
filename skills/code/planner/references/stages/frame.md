# Frame

- Identify the active repo, package, or service boundary.
- Restate the task as a concrete target: field, API, component, flag, schema, dependency, script, or workflow.
- State assumptions explicitly when they affect the plan.
- If multiple interpretations exist, present them instead of choosing silently.
- Preserve explicit constraints, no-touch areas, and success criteria.
- Record required or forbidden evidence sources and explicit domain guarantees, including states the user says cannot occur.
- Separate what is requested from what is actually needed to solve the problem.
- Define what will prove the task is complete.
- If the task target, constraints, or success criteria are still unclear, ask the user directly before continuing the plan.
