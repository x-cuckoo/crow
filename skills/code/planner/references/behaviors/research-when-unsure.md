# Research When Unsure

Use this when external evidence can materially improve a plan, especially for unfamiliar technology, dependency behavior, ecosystem convention, architecture tradeoffs, or current facts.

- Exhaust cheap local evidence first; research complements repo reading, never replaces it.
- Use web and community evidence early when it can improve the decision, not only after the plan is blocked.
- Prefer official docs, maintained open-source projects, release notes, issue discussions, and credible real-world examples over isolated opinions.
- Prefer mature, stable open source; for dependencies, choose the latest stable version compatible with the current runtime, package manager, dependency graph, and lockfile.
- Bring back concrete patterns, constraints, or examples that affect the decision.
